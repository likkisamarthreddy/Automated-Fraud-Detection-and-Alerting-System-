1. Transaction Source

Represents an external payment gateway, banking system, or merchant application.

Sends transaction details such as amount, time, location, and account information to the system for analysis.

Initiates the fraud detection process.

2. Fraud Analyst

A human expert responsible for reviewing fraud alerts generated by the system.

Investigates suspicious transactions and decides whether they are legitimate or fraudulent.

May take actions such as approving, blocking, or escalating transactions.

3. System Administrator

Responsible for monitoring system performance and reliability.

Manages audit logs, system configurations, and security settings.

Ensures the system is running smoothly and complies with organizational policies.

4. Developer

Uses the system for testing and validation purposes.

Simulates transaction events to test fraud detection logic and alert mechanisms.

Helps in debugging, enhancing, and maintaining the system.
