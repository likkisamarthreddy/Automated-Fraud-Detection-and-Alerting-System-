
## Use Case Diagram Theory

A Use Case Diagram is a UML diagram that represents the functional
requirements of a system from the user's perspective.

### Notations:
- Actor: External entity interacting with the system
- Use Case: System functionality
- System Boundary: Defines system scope
- Include: Mandatory sub-function
- Extend: Optional behavior under conditions

The diagram explains how transaction events are processed, analyzed,
and alerted upon when fraud is detected.
